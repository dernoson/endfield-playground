# 明日方舟：終末地｜集成工業模擬器
# Top Spec

**版本：** v0.3
**平台：** 純前端網頁（Vue 3 + Vite，無後端）
**目標受眾：** 一般玩家 ＋ 進階玩家（兩者兼顧）；本文件同時供開發同仁與 AI 輔助開發使用

---

## 1. 專案概述

提供一個忠實還原《明日方舟：終末地》集成工業系統的網頁模擬器，讓玩家可以在遊戲外預先規劃產線配置、即時估算產量與效率，並將結果匯出為可分享的記錄檔。

**核心價值：**
- **先試後蓋**：不用在遊戲內反覆拆裝，直接在網頁沙盒規劃
- **即時回饋**：每次擺放或連線後立即看到流量與效率估算
- **社群共享**：匯出記錄檔既可瀏覽分享，也可匯回繼續編輯

---

## 2. 技術選型

| 層次 | 技術 | 說明 |
|------|------|------|
| 框架 | **Vue 3 + Vite** | 團隊熟悉；Nuxt UI v3 支援純 Vue + Vite |
| UI 元件 | **Nuxt UI v3** | 125+ 元件，Sidebar、Table、Modal、Tooltip 等直接套用 |
| 狀態管理 | **Pinia** | Vue 官方推薦，與 vueUse 整合無縫 |
| 工具函式 | **vueUse** | `useMagicKeys` 管理快捷鍵、`useLocalStorage` 自動存檔 |
| 樣式 | **Tailwind CSS v4** | Nuxt UI v3 內建 |
| 畫布渲染 | **待 PoC 驗證** | 候選 A：Vue Flow（節點連線原生支援，需驗證格子制）；候選 B：Konva.js + vue-konva（完全掌控，開發量較大）|
| 資料格式 | **JSON**（設備庫）＋ **HTML**（記錄檔） | 見核心需求 6 |

> **畫布渲染 PoC 驗證目標：** snap-to-grid、多格設備佔位（3×3 精煉爐）、Port 連線，確認 Vue Flow 是否適用後再定案。

## 3. 核心需求概述

### CR-01　2D 格子制畫布與設備擺放
精確還原遊戲格線，設備依佔格數擺放。支援拖拉擺放、旋轉、移動，設備重疊時僅高亮警示不硬性阻擋。移動設備時，已連接管線維持連接但若產生衝突則顯示 Error；設備正對現有管線時，自動連接。點選工具列設備或畫布中已部署設備，左側面板顯示該設備的詳細資訊、配方表與設備大小。

支援選擇當前使用的基地（武陵 / 四號谷地，optional）；選擇後自動對應畫布大小。選擇武陵時，額外於畫布邊緣標示液體原物料的輸入方位。基地畫布不允許擺放中繼器。

框選設備與管線後按 Ctrl+C，滑鼠立即拖曳複製預覽組，點選確認擺放；框選範圍內的管線一併複製。

操作歷史由 CR-08 統一管理，Ctrl+Z / Ctrl+Y 可無限制復原 / 取消復原。

→ 詳見 `01_canvas_and_devices.md`

---

### CR-02　管線連接
切換至管線模式後，所有管線與連接點自動高亮。依連接點的 type 定義自動判斷拉的是水管或傳送帶，並自動吸附至連接點。以下情境自動處理：

- 從管線中途拉出新管線 → 自動生成分流器
- 將管線拉至已有管線中途 → 自動生成匯流器
- 路徑規劃時跨越現有管線 → 自動插入物流橋

**Phase 1：** 彎折點由使用者手動點選決定，確認兩端後可微調再放置。
**Phase 3：** 升級為自動路徑規劃，彎折點依當前空間自動生成。

→ 詳見 `02_pipeline.md`

---

### CR-03　產線設備與佈線警示
系統持續偵測配置狀態，分兩級警示：

- **Error**：設備重疊、佈線違法、配方不符、設備未在供電範圍內
- **Warning**：堵塞、輸入缺失、整區總耗電量超過總供電量上限

供電範圍以所有供電樁的 n×n 覆蓋格子取聯集計算，設備任一格在聯集內即視為供電成功。基地畫布不允許擺放中繼器。

警示列表顯示於右側統計面板，點選可導覽至畫布相應區域；相應設備或管線本身也顯示視覺警訊。有 Error 的設備與管線略過流量估算。

→ 詳見 `03_validation.md`

---

### CR-04　流量模擬與產能估算
每當設備擺放、管線變動、產物設定變更後，即時重新計算並在設備、管線、產線起點／終點顯示估算數值。右側統計面板顯示：

- 當前藍圖總耗電量與供電量比較
- 當前總產能估算（依品項列出生產 / 消耗 / 淨產量）
- 產能換算調度券兌換效率（兌換率由使用者自訂）

→ 詳見 `04_flow_simulation.md`

---

### CR-05　配方流程設計（Flow Chart 視角）
主畫面提供視角切換控項，位於編輯區左下角，Tab 鍵輪替，支援三種視角模式：

- **藍圖視角**：完整畫布，設備擺放與管線操作
- **流程視角**：Flow Chart 呈現產物流向、設備、效率與配比
- **並列視角**：藍圖與流程並排顯示，支援左右或上下並排（可切換）；點選任意一方另一方自動導覽至相應位置；任意一方編輯時另一方即時聯動變動

三種視角共用同一份產線資料，僅呈現方式不同。

流程視角功能：使用者可選擇中間產物的生成流程或直接從倉庫取用。新增流程配方後，系統顯示「切換視角以導入」提示；切回藍圖視角時，滑鼠自動拖曳一組對應設備，由使用者選擇擺放位置。

**Phase 1：** 三種視角切換上線（Tab 鍵 / 左下角圖示），可瀏覽現有產線的 Flow Chart 呈現；並列視角點選導覽聯動、支援上下 / 左右並排切換。
**Phase 2：** 新增流程配方、選擇中間產物流程、一鍵導入設備至藍圖視角；並列視角編輯聯動。

→ 詳見 `05_recipe_flow.md`

---

### CR-06　藍圖匯入 / 匯出
**匯出記錄檔（.html）** 內容包含：完整產線設備藍圖、流程 Flow Chart、產能估算、警示列舉。檔案分兩層：人類可讀層（靜態呈現 + 輕量互動）、機器可讀層（內嵌完整 JSON 供匯回）。

**匯入**支援 `.html` 與 `.json`，含跨版本 migrate 邏輯（以 Zod 實作，靜默執行）；僅在 parse error 時顯示錯誤訊息。

→ 詳見 `06_export_import.md`

---

### CR-07　多目標優化產能推導
獨立頁面。使用者設定調度券目標產能、部分產物的硬性產量需求、當前可用原料產能（以傳送帶條數／流量為單位），系統以連續線性規劃（LP）推導最佳配方流量分配，結果以「各配方分配幾條流量、對應幾台設備、運行效率」呈現。

使用者可微調台數或流量（兩者聯動：調台數反推流量、調流量換算建議台數並標示效率），完成後走「切換視角以導入」流程，進入擺放視角。

→ 詳見 `07_optimization.md`

---

### CR-08　操作歷史（History）
CR-01、CR-02、CR-05 的所有藍圖狀態變更統一由操作歷史 store 管理，採 Command Pattern 實作。支援無限制復原 / 取消復原（Ctrl+Z / Ctrl+Y），在藍圖視角、流程視角、並列視角任一處操作均進入同一歷史佇列。Session 結束後歷史清空，不持久化。

→ 詳見 `08_history.md`

## 4. 階段性目標

### Phase 1 — MVP（可用的產線規劃器 + 流程視角初體驗）
目標：能擺設備、拉管線、看到即時估算，並初步體驗流程視角。

- CR-01 基礎畫布與設備擺放（格子制、拖拉、旋轉、左側資訊面板、基地選擇、複製擺放）
- CR-02 管線連接（管線模式、自動吸附、分流器 / 匯流器 / 物流橋自動生成、手動彎折點）
- CR-03 Error 層級警示（含供電範圍偵測）
- CR-04 基礎流量估算（即時重算、右側統計面板、總耗電量顯示）
- CR-05 流程視角基礎（藍圖 / 流程 / 並列三種視角切換；並列視角點選導覽聯動）
- CR-08 操作歷史（無限制復原 / 取消復原）

### Phase 2 — 完整模擬體驗
目標：警示系統、流程規劃與分享功能完整上線。

- CR-03 Warning 層級警示
- CR-04 調度券兌換效率計算
- CR-05 新增流程配方、中間產物選擇、一鍵導入設備；並列視角編輯聯動
- CR-06 藍圖匯出（完整 HTML 記錄檔）、匯入與跨版本 migrate

### Phase 3 — 進階自動化
目標：自動佈線與產能優化。

- CR-02 自動路徑規劃（彎折點自動運算）
- CR-07 多目標優化產能推導（LP 求解、台數 / 流量微調、導入擺放視角）

## 5. 跨功能設計原則

- **資料單一來源**：畫布狀態、連線狀態、警示狀態統一存於 Pinia store，設備擺放視角與流程視角共用同一份資料
- **即時重算**：任何狀態變更後，FlowEngine 自動觸發重新計算（debounce 處理高頻操作）；有 Error 的節點略過估算
- **設備資料庫**：完整照搬遊戲所有設備與配方，以 `/data/devices.ts` 維護（TypeScript 定義，受型別系統保護）；每個接口含明確 type 定義（傳送帶 / 水管），供管線模式自動判斷
- **流量單位**：以傳送帶滿載流量（30/min）為基礎物流單位，設備台數為向上取整的結果
- **操作歷史單一佇列**：CR-01、CR-02、CR-05 的所有藍圖操作統一進入 `useHistoryStore`，任何視角的 Ctrl+Z / Ctrl+Y 均作用於同一歷史，Session 結束清空
- **自動存檔**：透過 vueUse `useLocalStorage` debounce 500ms 寫入，避免資料遺失
- **記錄檔自包含**：匯出 HTML 無外部依賴，離線可瀏覽；JSON 層完整，可無損匯回

## 6. 待確認事項

1. **設備圖示來源**：自繪 SVG 或遊戲截圖（需考量版權）
2. **設備資料庫維護責任**：由誰負責同步遊戲更新後的新設備 / 配方
3. **各協議核心區域的格子尺寸**：是否已有完整清點
4. **流量計算精度**：基準單位確認（個/秒 or 個/分鐘）、是否有浮點數精度問題
5. **畫布渲染 PoC**：Vue Flow 或 Konva.js，需實作驗證後定案
6. **調度券兌換率**：使用者自訂，是否需要支援多組預設值（不同商店 / 不同時期）
7. **LP 求解套件**：CR-07 建議使用 `glpk.js`，需確認授權與 bundle size 是否可接受

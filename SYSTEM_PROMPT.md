# 開發守則

關於專案使用 tech stack 與資料夾架構，請參考 README.md 文件。

此外，有一些守則需要特別注意：

## 程式碼註解規則

- 一律使用繁體中文 (除非是專有名詞)
- 嚴格禁止表情符號
- 使用 JSDoc 格式註解
- 註解目標包含：
    - 函數
    - 類別
        - 物件型態/介面型態，需要將成員各別註解說明
    - 全域變數
    - store
        - 內部使用 ref、computed、reactive 宣告的變數
        - 函數
        - return 輸出的各個成員，也需要註解，並且需要宣告的地方的註解一致
    - 大型頁面區段
    - .vue 中宣告的變數與函數
    - .vue 中宣告的 defineProps 與 defineEmits
    - .vue 中使用的副作用函數意義 (e.g. onMounted, watch, etc.)

## 程式碼設計規則

- 盡量使用 NuxtUI 的元件，而不是自己寫
- 盡量使用 VueUse 的函數，而不是自己寫
- 盡量使用 TailwindCSS 的內建 class，而不是自己寫
- 盡量遵照 SOLID 原則

## 注意事項

- 當開發者提出的需求會導致過於大量的程式碼變更，你必須提出與使用者討論的請求，規劃適當的步驟分割，一次僅執行一個步驟
- 你被嚴格禁止讀取 .env、.secrets 等敏感資料檔案
